## CRISPE框架

### 概览
CRISPE是一个全面的提示词框架，适用于复杂任务和需要精确控制的场景。它通过六个维度确保提示词的完整性和有效性。

### 框架组成

#### C - Capacity/Role (能力/角色)
**定义**：指定AI应该扮演的角色和具备的专业能力

**要点**：
- 使用职业身份（工程师、医生、律师等）
- 指定经验年限和专业领域
- 明确角色特征（耐心、严谨、创新等）

**示例**：
- "你是一位具有15年经验的资深数据科学家"
- "你是一名专注于用户体验的产品设计师"
- "你是一位善于解释复杂概念的计算机科学教授"

**为什么重要**：角色聚焦AI的响应视角，激活相关知识域，统一回答风格

#### R - Request (请求)
**定义**：明确告诉AI要执行什么任务

**要点**：
- 使用具体行动动词
- 避免模糊表述
- 明确任务的边界

**示例**：
- "分析这份用户反馈报告，提取主要情感倾向"
- "设计一个Python函数来处理时间序列数据"
- "撰写一篇关于机器学习入门的技术博客"

**为什么重要**：清晰的请求是AI理解任务的核心，直接影响输出质量

#### I - Information (信息)
**定义**：提供AI完成任务所需的背景信息

**要点**：
- 提供足够的上下文
- 包含项目细节、目标、约束
- 避免假设AI知道任何事情

**示例**：
- "我们是一家B2B SaaS公司，主要客户是中小企业，使用Node.js开发后端服务"
- "这个API用于处理支付请求，每秒约处理1000次请求，需要保证高可用性"
- "目标受众是产品经理，他们有一定的技术背景但不是开发者"

**为什么重要**：AI没有记忆和访问用户文件的能力，所有相关信息必须在提示词中提供

#### S - Situation (情境)
**定义**：描述当前的具体情况和环境

**要点**：
- 说明当前阶段或状态
- 描述面临的具体挑战
- 解释时间压力或其他限制

**示例**：
- "我们正在准备下周的产品发布会，时间紧迫"
- "当前系统存在性能瓶颈，需要紧急优化"
- "这是项目的第一个原型，优先验证核心功能而非完整性"

**为什么重要**：情境信息帮助AI调整响应的优先级和策略

#### P - Persona (人设)
**定义**：指定AI的沟通风格和语气

**要点**：
- 确定正式程度（正式/非正式）
- 选择语言风格（技术性/通俗/幽默）
- 指定文化或地域特征

**示例**：
- "使用友好、鼓励的语气，像一位耐心的导师"
- "保持专业、客观的学术风格"
- "使用轻松、幽默的网络流行语风格"

**为什么重要**：人设确保输出符合用户的期望和品牌调性

#### E - Experiment (实验/示例)
**定义**：提供输入输出的示例来演示期望的行为

**要点**：
- 提供1-3个清晰示例
- 展示输入到输出的映射
- 说明期望的格式和质量

**示例**：
```
示例输入：
"登录页面加载缓慢"

示例输出：
**性能问题诊断**
- 问题类型：页面加载性能
- 可能原因：资源文件过大、网络延迟、服务器响应慢
- 优先级：高
- 建议方案：优化图片压缩、使用CDN、检查数据库查询
```

**为什么重要**：示例是最强大的指导方式，通过少样本学习让AI精确理解期望

### CRISPE完整示例

```markdown
你是一位具有10年经验的资深营销文案撰写人（Capacity）。

请为我们的新产品撰写一份产品介绍文案（Request）。

背景信息：我们是一家初创公司，开发了一款AI驱动的个人知识管理工具。目标用户是知识工作者、研究人员和学生。产品核心功能包括：智能笔记整理、自动关联相关内容、快速搜索、多设备同步。我们的品牌调性是专业但不过于严肃（Information）。

当前情境：产品即将在下个月正式发布，这是首次向公众介绍我们的产品。我们需要在LinkedIn和Twitter上发布，希望能够在专业社群中引起关注（Situation）。

沟通风格：使用专业但亲和的语气，避免过度营销语言，以价值导向而非功能罗列为主（Persona）。

示例风格：
不要写："我们的产品有智能笔记整理功能"
要写："让你的知识自动形成网络，轻松发现隐藏的关联"（Experiment）
```

## RTF框架

### 概览
RTF是一个简洁高效的框架，适用于中等复杂度的任务。它强调三个核心要素：角色、任务和格式。

### 框架组成

#### R - Role (角色)
**定义**：AI应该扮演的角色

**要点**：
- 简洁明确
- 与任务相关
- 包含必要的专业背景

**示例**：
- "你是一位Python开发专家"
- "你是一位创意写作教练"
- "你是一位心理咨询师"

#### T - Task (任务)
**定义**：AI需要执行的具体任务

**要点**：
- 使用行动动词
- 明确任务范围
- 指定期望的深度

**示例**：
- "审查以下代码，识别3个可以改进的地方"
- "为这个故事写一个500字的续集"
- "提供5个缓解工作压力的方法"

#### F - Format (格式)
**定义**：期望的输出格式

**要点**：
- 明确结构
- 指定长度
- 说明呈现方式

**示例**：
```
- 使用markdown格式
- 分为3个部分：概述、分析、建议
- 总长度不超过500字
```

### RTF完整示例

```markdown
Role: 你是一位具有8年经验的前端开发工程师，精通React和性能优化。

Task: 审查以下React组件代码，识别性能问题和优化机会。

Format:
请按以下格式返回：
## 性能问题
列出发现的性能问题，每个问题包括：
- 问题位置（代码行号）
- 问题类型（重复渲染、内存泄漏、不必要的计算等）
- 严重程度（高/中/低）

## 优化建议
为每个问题提供具体的优化方案，包括：
- 解决方案描述
- 预期改进效果
- 实施难度

## 代码示例
提供一个优化后的代码片段展示改进方法
```

## RISE框架

### 概览
RISE框架专注于系统化推理和结构化思维，适用于需要深入分析和逻辑推理的任务。

### 框架组成

#### R - Role (角色)
**定义**：执行任务的专业角色

#### I - Input (输入)
**定义**：描述信息或资源

**示例**：
- "提供以下用户反馈数据：[数据]"
- "基于以下市场分析报告：[报告]"
- "参考以下技术文档：[文档]"

#### S - Steps (步骤)
**定义**：将复杂任务分解为可执行的步骤

**示例**：
```
步骤1：定义问题边界
步骤2：收集关键信息
步骤3：应用分析框架
步骤4：得出结论
步骤5：验证结果
```

#### E - Expectation (期望)
**定义**：描述所需的结果

**示例**：
- "提供明确的决策建议"
- "给出详细的优化方案"
- "输出结构化的分析报告"

### RISE完整示例

```markdown
Role: 你是一位战略咨询顾问，擅长商业分析和决策支持。

Input: 我需要你帮助评估是否应该为我们的产品添加一项新功能。

Steps:
1. 问题定义：明确要评估的功能和核心决策问题
2. 信息收集：列出需要考虑的关键因素（用户需求、技术可行性、成本、竞争环境等）
3. 利益相关者分析：识别所有利益相关者及其关切点
4. 方案评估：分析支持与反对的论据，权衡利弊
5. 风险评估：识别潜在风险和缓解措施
6. 最终建议：基于分析提出明确的建议和实施计划

Expectation: 提供一个结构化的决策分析报告，包含明确的建议和风险评估。
```

## 其他框架

### A.P.E框架

#### 概览
APE是一个简洁的三要素框架，适用于快速构建基本提示词。

#### 框架组成

**A - Action (行动)**
- 定义：定义要完成的工作或活动
- 示例："分析这个数据集"
- 要点：使用明确的行动动词

**P - Purpose (目的)**
- 定义：讨论意图或目标
- 示例："目的是找出销售趋势"
- 要点：明确为什么要做这件事

**E - Expectation (期望)**
- 定义：陈述预期的结果
- 示例："期望得到3-5个关键发现"
- 要点：具体描述期望的输出

#### APE示例
```markdown
Action: 分析这个月的销售数据

Purpose: 目的是识别销售趋势和异常点，为下个月的营销策略提供依据

Expectation: 期望得到：
- 销售趋势的3-4个关键发现
- 识别的2-3个异常数据点
- 基于发现的2个具体建议
```

### B.R.O.K.E框架

#### 概览
BROKE是一个全面的五要素框架，特别适合需要明确目标和结果的场景。

#### 框架组成

**B - Background (背景)**
- 定义：说明背景，提供充足信息
- 示例："我们是一家电商公司，年销售额5000万"
- 要点：提供充分的上下文信息

**R - Role (角色)**
- 定义：我希望ChatGPT扮演的角色
- 示例："你是一位市场营销专家"
- 要点：明确AI的角色定位

**O - Objectives (目标)**
- 定义：我们希望实现什么
- 示例："目标是提升品牌知名度和用户参与度"
- 要点：清晰的目标陈述

**K - Key Result (关键结果)**
- 定义：我要什么具体效果
- 示例："社交媒体关注量增长50%"
- 要点：可量化的结果指标

**E - Evolve (试验并改进)**
- 定义：三种改进方法自由组合
- 示例："测试不同的内容策略，根据数据反馈调整"
- 要点：强调迭代优化

#### BROKE示例
```markdown
Background: 我们是一家B2B SaaS公司，提供项目管理工具，当前月活用户2万，正在准备进入北美市场。

Role: 你是一位具有10年经验的市场营销总监，擅长B2B SaaS产品的市场进入策略。

Objectives: 帮助我们制定北美市场的进入策略，目标是6个月内获得1000个付费用户。

Key Result: 期望得到：
- 详细的市场进入策略
- 3个核心营销渠道
- 预估的获客成本和预算分配
- 首月、3个月、6个月的目标里程碑

Evolve: 为策略提供2-3个备选方案，并说明如何根据初期数据调整策略。
```

### C.O.A.S.T框架

#### 概览
COAST是一个五要素框架，强调场景和行动，适合任务导向的场景。

#### 框架组成

**C - Context (上下文)**
- 定义：为对话设定舞台
- 示例："我们正在开发一个移动应用"
- 要点：提供基本的情境信息

**O - Objective (目的)**
- 定义：描述目标
- 示例："目标是设计用户注册流程"
- 要点：明确任务目标

**A - Action (行动)**
- 定义：解释所需的动作
- 示例："设计一个简单快速的注册界面"
- 要点：说明具体要做什么

**S - Scenario (场景)**
- 定义：描述场景
- 示例："用户首次打开APP的场景"
- 要点：提供具体的使用场景

**T - Task (任务)**
- 定义：描述任务
- 示例："设计UI流程和交互细节"
- 要点：明确的任务描述

#### COAST示例
```markdown
Context: 我们正在开发一款健身追踪APP，目标用户是25-35岁的都市白领。

Objective: 设计用户首次打开APP时的引导流程。

Action: 设计一个简洁、吸引人的新手引导界面，让用户快速了解核心功能并完成首次设置。

Scenario: 用户下载APP后首次打开，对APP功能一无所知，需要在30秒内建立对产品的第一印象。

Task: 设计包含以下内容的引导流程：
- 3-4个关键功能介绍页面
- 简洁的设置向导（个人资料、目标设定）
- 鼓励用户完成首次记录
```

### R.O.S.E.S框架

#### 概览
ROSES是一个五要素框架，结构完整，适合需要系统性解决方案的场景。

#### 框架组成

**R - Role (角色)**
- 定义：指定ChatGPT的角色
- 示例："你是一位产品经理"
- 要点：角色定位

**O - Objective (目的)**
- 定义：陈述目标或目标
- 示例："目标是提升用户留存率"
- 要点：目标陈述

**S - Scenario (场景)**
- 定义：描述情况
- 示例："当前月流失率15%"
- 要点：场景描述

**E - Expected Solution (解决方案)**
- 定义：定义所需的结果
- 示例："期望提出3个具体方案"
- 要点：解决方案定义

**S - Steps (步骤)**
- 定义：要求达到解决方案所需的措施
- 示例："分步骤说明实施方法"
- 要点：实施步骤

#### ROSES示例
```markdown
Role: 你是一位具有8年经验的产品经理，擅长用户增长和留存策略。

Objective: 帮助我们提升移动APP的用户留存率，当前30天留存率为15%，目标提升到30%。

Scenario: APP是一款社交健身应用，主要用户是25-35岁的都市白领，目前日活5万，月活20万。用户在第7天流失率最高。

Expected Solution: 期望提供：
- 用户流失原因的深度分析
- 3个具体的留存提升方案
- 每个方案的预估效果和实施成本
- 优先级排序

Steps: 对于每个方案，请提供：
1. 方案的核心思路
2. 具体实施步骤
3. 所需资源和时间
4. 成功指标和评估方法
```

### I.C.I.O框架

#### 概览
ICIO是一个四要素框架，特别适合数据处理和转换任务。

#### 框架组成

**I - Instruction (指令)**
- 定义：AI执行的具体任务
- 示例："转换数据格式"
- 要点：明确指令

**C - Context (背景)**
- 定义：给AI更多的背景信息
- 示例："这是客户反馈数据"
- 要点：背景信息

**I - Input Data (输入数据)**
- 定义：告知模型需要处理的数据
- 示例："输入CSV格式的数据"
- 要点：数据说明

**O - Output Indicator (输出引导)**
- 定义：告知我们要输出的类型或风格
- 示例："输出为JSON格式"
- 要点：输出格式

Instruction: 将用户反馈数据转换为结构化的分析报告，识别主要问题和趋势。

### R.A.C.E框架

#### 概览
RACE是一个四要素框架，简洁而完整，适合大多数常见任务。

#### 框架组成

**R - Role (角色)**
- 定义：指定ChatGPT的角色
- 示例："你是一位技术文档撰写人"
- 要点：角色定义

**A - Action (行动)**
- 定义：详细说明需要采取什么行动
- 示例："撰写API文档"
- 要点：行动说明

**C - Context (上下文)**
- 定义：提供有关情况的相关细节
- 示例："这是RESTful API文档"
- 要点：上下文信息

**E - Expectation (期望)**
- 定义：描述预期结果
- 示例："期望得到清晰的开发者文档"
- 要点：结果期望

#### RACE示例
```markdown
Role: 你是一位具有10年经验的技术文档撰写人，专门为开发者编写API文档。

Action: 为以下API端点编写完整的开发者文档。

Context: 这是一个支付处理服务的API，使用RESTful架构。目标读者是集成我们API的外部开发者，他们有基本的HTTP和JSON知识。文档将发布在我们的开发者门户上。

Expectation: 期望文档包含：
- 端点概述（功能说明、用途）
- 请求详情（方法、URL、Headers、Body参数及示例）
- 响应详情（成功响应、错误响应及示例）
- 代码示例（Python和JavaScript）
- 常见问题和使用注意事项

文档风格：
- 清晰简洁，避免过度技术化
- 使用Markdown格式
- 包含可直接运行的代码示例
```

## 框架对比与选择

### 框架复杂度分级

**简单框架（3要素）**：
- A.P.E
- T.A.G
- E.R.A
- 适合：快速任务、简单请求、日常使用

**中等框架（4-5要素）**：
- R.T.F
- R.A.C.E
- C.A.R.E
- I.C.I.O
- 适合：常规任务、技术工作、明确场景

**复杂框架（5-6要素）**：
- C.R.I.S.P.E
- B.R.O.K.E
- C.O.A.S.T
- T.R.A.C.E
- R.I.S.E
- R.O.S.E.S
- 适合：复杂项目、系统化分析、重要决策

### 按场景选择框架

#### 创意写作
- 推荐：C.O.A.S.T（强调场景）
- 备选：C.R.I.S.P.E（完整控制）

#### 技术开发
- 推荐：R.T.F（简洁有效）
- 备选：T.R.A.C.E（包含示例）

#### 数据处理
- 推荐：I.C.I.O（专为数据设计）
- 备选：T.A.G（快速处理）

#### 决策分析
- 推荐：R.I.S.E（强调推理）
- 备选：B.R.O.K.E（关注结果）

#### 商业策略
- 推荐：R.O.S.E.S（系统性）
- 备选：B.R.O.K.E（目标导向）

#### 快速任务
- 推荐：T.A.G（最简洁）
- 备选：A.P.E（三要素）

#### 产品设计
- 推荐：C.O.A.S.T（场景驱动）
- 备选：C.R.I.S.P.E（全面控制）

#### 营销文案
- 推荐：C.R.I.S.P.E（人设控制）
- 备选：C.A.R.E（强调结果）

### 使用建议

1. **初学者路径**：
   - 第1阶段：掌握R.T.F（经典框架）
   - 第2阶段：学习C.R.I.S.P.E（全面框架）
   - 第3阶段：根据需求选择其他专业框架

2. **效率优先**：
   - 日常快速任务：T.A.G或A.P.E
   - 技术工作：R.T.F
   - 数据处理：I.C.I.O

3. **质量优先**：
   - 重要项目：C.R.I.S.P.E或B.R.O.K.E
   - 复杂决策：R.I.S.E或R.O.S.E.S
   - 系统方案：R.O.S.E.S

4. **灵活应用**：
   - 可以混合不同框架的元素
   - 根据任务调整框架
   - 不要拘泥于框架形式

5. **进阶技巧**：
   - 创建个人模板库
   - 记录有效模式
   - 迭代优化框架应用

记住：框架是工具，不是枷锁。选择最适合你的框架，或灵活组合不同框架的元素，以达到最佳效果。
